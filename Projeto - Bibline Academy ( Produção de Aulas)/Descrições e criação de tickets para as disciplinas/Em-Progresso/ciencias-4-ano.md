> **⚠️ IMPORTANTE — Campo `# Description` na criação dos tickets:**
> Ao criar tickets no Kanban do GitHub, o corpo de cada ticket DEVE conter obrigatoriamente o cabeçalho `# Description` seguido da descrição do conteúdo. Este campo faz parte da estrutura padrão do ticket e NÃO pode ser omitido.
>
> **Exemplo de estrutura do ticket:**
> ```
> [Ciências] - Ano 4 - 11.4 Revisão
>
> # Description
>
> Revisão dos conhecimentos das 3 aulas anteriores
> ```
>
> Esta regra vale para TODOS os tickets: aulas regulares, revisões (.4), provas (.5), revisões bimestrais e provas bimestrais.
