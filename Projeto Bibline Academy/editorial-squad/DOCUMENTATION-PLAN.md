## 📝 Template de Documentação

Cada agente deve seguir este template:

```markdown
# 🎯 [Nome do Agente] - Documentação Completa

## Visão Geral
- **Nome:** [Nome do agente]
- **Tipo:** [Categoria]
- **Status:** ✅ Ativo / ⚠️ Legacy / 🚧 Em desenvolvimento
- **Versão:** 1.0

## Objetivo
[Descrição de 1-2 parágrafos]

## Comandos Disponíveis

### `/comando <parametro>`
**Descrição:** [O que o comando faz]

**Sintaxe:**
```bash
/comando <parametro> [opcional]
```

**Parâmetros:**
| Nome | Tipo | Obrigatório | Descrição |
|------|------|-------------|-----------|
| `parametro` | string | ✅ | Descrição |

**Exemplo:**
```bash
/comando valor
```

**Saída:**
[O que o comando retorna]

## Métricas
[Quais métricas o agente usa/gera]

## Estrutura de Relatórios
[Como os relatórios são estruturados]

## Casos de Uso
[Quando e como usar o agente]

## Integrações
[Com quais sistemas o agente se integra]

## FAQ
[Perguntas frequentes]

## Links Úteis
- [Skill Definition](SKILL.md)
- [Knowledge Base](../knowledge-base/)
- [Templates](../templates/)
```

## ✅ Checklist de Qualidade

Cada documentação deve ter:

- [ ] Visão geral clara e objetiva
- [ ] Todos os comandos listados com exemplos
- [ ] Parâmetros documentados em tabelas
- [ ] Exemplos de uso prático
- [ ] Métricas explicadas
- [ ] Casos de uso reais
- [ ] Integrações listadas
- [ ] FAQ com perguntas comuns
- [ ] Links para recursos relacionados
- [ ] Formatação consistente (Markdown)
- [ ] Revisão ortográfica
- [ ] Validação técnica

## 🔄 Processo de Revisão

### **1. Auto-Revisão**

Antes de submeter:

- [ ] Verificar ortografia e gramática
- [ ] Testar todos os exemplos de comando
- [ ] Validar links internos e externos
- [ ] Confirmar formatação Markdown

---

### **2. Revisão por Pares**

Designar 1-2 revisores:

- [ ] Revisor técnico (conhece o agente)
- [ ] Revisor de conteúdo (foco em clareza)
- [ ] Revisor de UX (foco em usabilidade)

---

### **3. Validação Final**

- [ ] Aprovação do responsável pela skill
- [ ] Aprovação do gestor da squad
- [ ] Merge para main branch
- [ ] Publicação no repositório
